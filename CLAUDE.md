# CLAUDE.md

이 저장소는 코드 저장소가 아니라 소설 **『무소유(無所有)』 설정 바이블**이다.
세션 시작 시 이 문서를 먼저 읽고, 아래 원칙을 따른다.

## 저장소 성격

- 산문/설정 자료를 다루는 저장소. 빌드·테스트·린트 없음.
- **진실의 유일한 기준은 이 저장소의 md 파일들이다. 대화 기록에는 의존하지 않는다.**
- 핵심 주제: "착한 사람은 먼저 지친다 — 성실함은 손해가 되는 곳에서." 구조가 착한 사람에게 청구서를 몰아주기 때문에 먼저 지친다는 것이 제목의 완성.

## 문서 구조

| 파일 | 내용 |
|------|------|
| `world.md` | 세계의 제1원리, 불변의 상수, 세계관, 빌런 분류 축 |
| `narrator.md` | 화자의 역설·레이어 구조·기원 서사 |
| `characters/README.md` | 인물 인덱스 |
| `characters/villain-1~4.md` | 빌런 4인 (①은 미확정) |
| `characters/foil-1~3.md` | 대비 인물 (③은 후보, 관찰 중) |
| `characters/manager.md` | 관리자 |
| `episodes.md` | EP.1~4 |
| `archive.md` | 시각 자료 아카이브 |
| `timeline.md` | 전체 연대표 (시점 검증용) |
| `tone_guide.md` | 톤 가이드 + 기록 원칙 |
| `title.md` | 제목 확정 사항 |
| `editorial.md` | 구조 진단·선별 기준·집필 위험 관리·미반영 재료 대기열 |

**파일명 정책:** 인물 파일명은 실명 대신 코드네임(villain-N, foil-N, manager)을 쓴다. 실명은 각 파일 내부 콘텐츠에만 존재 — 리포 외부 노출(검색·링크·디렉토리 목록)을 줄이기 위함. 새 인물 파일을 만들 때도 이 정책을 따른다.

## 운영 원칙 (README.md 기준)

1. 저장소의 md 파일이 최우선 기준. 대화는 저장소가 아니다.
2. 문서에 없는 내용은 사실처럼 단정하지 않는다. 애매한 것은 추정/후보/미확정으로 표시.
3. 새 설정이 기존 문서와 충돌하면 충돌 지점을 명시한다.
4. 새 설정 추가 시 기존 설정과의 관계를 함께 적는다.
5. 인물/세계관/에피소드/시간선/톤가이드는 분리 관리한다.
6. 원문 의미를 바꾸는 "보완"은 하지 않는다. 중요한 변경은 전/후를 함께 제시한다.
7. 변경은 GitHub 커밋 단위로 작게 쪼갠다.

## 집필/편집 시 주의사항 (editorial.md 기준)

- **재료 판단 기준:** 성실한 자가 손해 보는 구조를 보여주거나, 화자가 그것을 읽어내는 눈의 기원을 설명하면 → 안쪽(반영). 둘 다 아니면 → 바깥(버리지 않고 광산에 보관, 연결 생기면 편입).
- 입력 단계에서 자기검열 불필요. 선별은 나중 일.
- **파일 생성/반영은 명시적 요청("반영해줘" 등)이 있을 때만 수행한다. 평시는 대화로 분석만 한다.**
- 정기 전체 검토는 "전체 검토해줘" 요청 시 수행 — 상호 참조·날짜 교차 검증 후 README.md 검토 로그에 기록.
- 단어·표현에 대한 과잉 해석 금지. 정리되지 않은 표현이 와도 표현 자체를 분석하지 않고 뒤의 사실과 흐름만 붙잡는다.
- **감정/날것 표현 불간섭.** 초고 단계 감정적 단어, 인신공격적 표현은 다듬거나 딴지 걸지 않고 그대로 기록한다. 반복 지적 금지.
- 배경 인물(핵심 시트 없는 다수)에게 함부로 서사를 부여하지 않는다 — 핵심 인물(빌런4+관리자4+대비3+화자)과 배경 인물(timeline.md 명단)은 2계층으로 분리 유지.

## 세션 워크플로우

1. 작업 시작 전 `README.md`와 `editorial.md`를 확인해 최신 운영 원칙/대기열을 파악한다.
2. 새 자료 반영 요청 시 어느 파일에 속하는지 문서 구조표를 기준으로 판단한다.
3. 반영 완료 후 `editorial.md`의 "반영 완료 로그"(또는 관련 로그 섹션)에 날짜와 내용을 남긴다.
4. 전체 검토 수행 시 `README.md`의 "전체 검토 로그"에 결과를 기록한다.
