## 운영 원칙 (2026-07-09 확정)

1. 이 저장소의 md 파일이 최우선 기준. 대화는 저장소가 아니다.
2. 문서에 없는 내용은 사실처럼 단정하지 않는다. 애매한 것은 추정/후보/미확정으로 표시한다.
3. 새 설정이 기존 문서와 충돌하면 충돌 지점을 명시한다.
4. 새 설정 추가 시 기존 설정과의 관계를 함께 적는다.
5. 인물/세계관/에피소드/시간선/톤가이드는 분리 관리한다 (본 구조).
6. 원문 의미를 바꾸는 "보완"은 하지 않는다. 중요한 변경은 전/후를 함께 제시한다.
7. 변경은 GitHub 커밋 단위로 작게 쪼갠다.
